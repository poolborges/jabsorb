JSON-RPC-Java

JSON-RPC-Java is a Java implementation of the JSON-RPC protocol.

Introduction

JSON-RPC-Java is a dynamic JSON-RPC to Java bridge that allows you to transparently call server-side Java code from Javascript.

There should be no changes necessary to your existing server-side Java code to allow calling from JavaScript (such as marshalling and unmarshalling of special types) as JSON-RPC-Java dynamically maps JavaScript Objects to Java Objects by dynamic reflection on Java method signatures and dynamic mapping between Java Objects and JavaScript objects.

It maps the loosely typed JavaScript objects to Java objects by reflecting on the Java method signatures and matching the fields of JSON objects to Java primitive types, arrays and complex objects such as Java Beans.

It currently supports marshalling and unmarshalling of the following types:

  • Primitive types (int, long, short, byte, char, float, double)
  • Numbers (Float, Integer, etc...), Strings, Char and Byte arrays
  • Java Beans (any Object with set and get methods)
  • Arrays of primitive types, Strings, Numbers, Collections and Java Beans
  • Exceptions (although type information isn't currently preserved)
  • Concrete and abstract collection types:
    • AbstractList, ArrayList, LinkedList, and Vector
    • AbstractMap, HashMap, TreeMap, and LinkedHashMap
    • AbstractSet, HashSet, TreeSet, and LinkedHashSet
    • Dictionary, Hashtable
  • Any arbitrary nested combination of the above

Bugs and Limitations

  • Support is currently limited to Netscape 7, latest Safari, newer versions of Mozilla, and Internet Explorer 5 and up. The jsolait jsonrpc client library does not work on IE 5.1 for Mac, Netscape 6, Opera and Konqueror due to their lack of support for the XMLHttpRequest object.
  • Implementation of a Java Applet to provide synchronous communication on browsers with Java VM and Javascript to Java Bridge (Konqueror and Netscape 6) will extend the supported set of browsers. I currently have an experimental implementation of this (with changes to urllib in jsolait) which you can mail me if you're interested.

Please send bug reports to michael@metaparadigm.com
